É difícil aprender?

Muitos livros que ensinam HTML e CSS lembram manuais sem graça. Para facilitar seu aprendizado, jogamos fora o modelo tradicional usado por editoras e redesenhamos este livro do zero.

Entender HTML e CSS pode ajudar qualquer pessoa que trabalhe com a web, designers podem criar sites mais atraentes e úteis, editores de sites podem criar melhor conteúdo, profissionais de marketing podem se comunicar com seu público de forma mais eficaz e gerentes podem comissionar sites melhores e obter o melhor de suas equipes.

Eu me concentrei no código que você precisa usar 90% do tempo e omiti o código que você raramente veria, mesmo que escrever sites fosse seu trabalho em tempo integral. No final do livro, se você encontrar outros 10%, poderá pesquisar no Google para descobrir o que eles significam de forma rápida e fácil.

Também adicionei informações práticas sobre tópicos que me são frequentemente questionados, como preparar imagens, áudio e vídeo para a web, como abordar o design e a construção de um novo site, como melhorar sua classificação em mecanismos de busca (SEO) e como usar o Google Analytics para aprender sobre palavras-chave para o seu site.

Como as pessoas acessam a web?

Antes de analisarmos o código usado para criar sites, é importante considerar as diferentes maneiras pelas quais as pessoas acessam a web e esclarecer algumas terminologias.

Navegadores

As pessoas acessam sites usando um software chamado navegador da web. Exemplos populares incluem Firefox, Edge, Safari, Chrome e Opera.

Para visualizar uma página da web, os usuários podem digitar um endereço da web no navegador, seguir um link de outro site ou usar um favorito.

Os fabricantes de software regularmente lançam novas versões de navegadores com novos recursos e suporte a novas adições de idiomas. É importante, no entanto, lembrar que muitos proprietários de computadores não estão executando as versões mais recentes desses navegadores. Portanto, você não pode confiar que todos os visitantes do seu site serão capazes de usar a funcionalidade mais recente oferecidas em todos os navegadores.

Você aprenderá como saber quais navegadores os visitantes usam para acessar o seu site no capítulo 19

Servidores Web

Quando você solicita uma página da web ao seu navegador, a solicitação é enviada pela internet para um computador especial conhecido como servidor web, que hospeda o site.

Servidores web são computadores especiais que estão constantemente conectados à internet e são otimizados para enviarem páginas da web para pessoas que as solicitam.

Algumas grandes empresas administram seus próprios servidores web, mas é mais comum usarem os serviços de uma empresa de hospedagem web que cobra uma taxa para hospedar seu site.

Dispositivos

As pessoas estão acessando sites em uma gama cada vez maior de dispositivos, incluindo computadores de mesa, laptops, tablets e celulares. É importante lembrar que vários dispositivos têm tamanhos de tela diferentes e alguns têm conexões mais rápidas com a web do que outros.

Leitores de tela

Leitores de tela são programas que leem o conteúdo da tela de um computador para um usuário. Eles são comumente usados por pessoas com deficiência visual.

Da mesma forma que muitos países têm legislações que exigem que prédios públicos sejam acessíveis a pessoas com deficiência, muitas leis também foram aprovadas exigindo que sites sejam acessíveis a pessoas com deficiência.

Ao longo deste livro, você verá várias referências a leitores de tela. Essas notas ajudaram a garantir que os sites que você criar sejam acessíveis a pessoas que usam esse software.

É interessante notar que tecnologias semelhantes às empregadas pelos leitores de tela também estão sendo usadas em outras áreas onde as pessoas não conseguem ler uma tela, como quando estão dirigindo ou correndo.

Como os sites são criados?

Todos os sites usam HTML e CSS, mas sistemas de gerenciamento de conteúdo, software de blogs e plataformas de comércio eletrônico geralmente adicionam mais algumas tecnologias à mistura.

O que você vê

Quando você está olhando para um site, é provável que seu navegador esteja recebendo HTML e CSS do servidor web que hospeda o site. O navegador web interpreta o código HTML e CSS para criar a página que você vê.

A maioria das páginas da web também inclui conteúdo extra, como imagens, áudio, vídeos ou animações e este livro lhe ensinará como prepará-los para uso na web, em seguida, como inseri-los em suas páginas web.

Alguns sites também enviam JavaScript ou Flash para o seu navegador, e você verá como adicionar JavaScript e Flash nas suas páginas web. Ambas as tecnologias são tópicos avançados sobre os quais você pode aprender mais após dominar HTML e CSS, se você quiser.

Como isso é criado

Pequenos sites são escritos geralmente usando apenas HTML e CSS.

Sites maiores, em particular aqueles que são regularmente atualizados e usam um sistema de gerenciamento de conteúdo (CMS), ferramentas de blog ou software de e-commerce, geralmente usam tecnologias mais complexas no servidor web, mas essas tecnologias são usadas para produzir HTML e CSS que são então enviados ao navegador. Então, se seu site usa essas tecnologias, você poderá usar seu conhecimento de HTML e CSS para ter mais controle sobre a aparência do seu site.

Sites maiores e mais complexos como esses podem usar um banco de dados para armazenar dados e linguagens de programação como PHP, ASP.Net, Java ou Ruby no servidor web, mas você não precisa conhecer essas tecnologias para melhorar o que o usuário vê. As habilidades que você aprenderá neste livro devem ser suficientes para ajudá-lo nessa jornada.

HTML5 e CSS3

Desde que a web foi criada, houve diversas versões de HTML e CSS, cada uma delas com a intenção de ser uma melhoria em relação à versão anterior.

No momento em que este livro foi escrito, HTML5 e CSS3 ainda estavam sendo desenvolvidos. Embora não tivessem sido finalizados, muitos navegadores já estavam suportando alguns recursos dessas linguagens e muitas pessoas estavam usando o código mais recente em seus sites. Portanto, escolhi ensinar a você essas versões mais recentes.

Como HTML5 E CSS3 são construídas sobre versões anteriores dessas linguagens, aprender isso significa que você também será capaz de entender as versões anteriores delas. Adicionei notas claras quando o código é novo e também quando ele pode não funcionar em navegadores mais antigos.

Como a web funciona

Quando você visita um site, o servidor web que hospeda este site pode estar em qualquer lugar do mundo. Para você encontrar a localização do servidor web, seu navegador primeiro se conectará a um servidor Domain Name System (DNS).

Nesta página você pode ver exemplos que demonstram como o servidor web que hospeda o site que você está visitando pode estar em qualquer lugar do mundo. São os servidores DNS que dizem ao seu navegador como encontrar o site.

  • Um usuário em Barcelona visita sony.jp in Tokyo
  • Um usuário em Nova Iorque visita google.com em San Francisco
  • Um usuário em Estocolmo visita qantas.com.au em Sydney
  • Um usuário em Vancouver visita airindia.in em Bangalore

À direita, você pode ver o que acontece quando um usuário da web na Inglaterra quer visualizar o site da galeria de arte do Louvre na França, que está localizado em www.louvre.fr. Primeiro, o navegador em Cambridge contata um servidor DNS em Londres. O servidor DNS então informa ao navegador a localização do servidor web que hospeda o site em Paris.

1 Quando você se conecta à web, você o faz por meio de um Provedor de Serviços de Internet (ISP). Você digita um nome de domínio ou endereço da web no seu navegador para visitar o site, por exemplo: google.com, bbc.co.uk, microsoft.com.

2 Seu computador contata uma rede de servidores chamados servidores Domain Name System (DNS). Eles agem como listas telefônicas; eles informam ao seu computador o endereço IP associado ao nome de domínio solicitado. Cada dispositivo na web tem um endereço IP exclusivo; é como o número de telefone daquele computador. Tradicionalmente, esses eram números de até 12 dígitos separados por pontos finais, mas agora estão sendo atualizados para conjuntos de até 32 caracteres.

3 O número exclusivo que o servidor DNS retorna ao seu computador permite que seu navegador entre em contato com o servidor web que hospeda o site que você solicitou. Um servidor web é um computador que está constantemente conectado à web e é configurado especialmente para enviar páginas da web aos usuários.

4 O servidor web então envia a página que você solicitou de volta para o seu navegador.
