Texto

Ao criar uma página da web, você adiciona tags (conhecidas como marcação) ao conteúdo da página. Essas tags fornecem significado extra e permitem que os navegadores mostrem ao usuário a estrutura apropriada para a página.

Neste capítulo, focamos em como adicionar marcação ao texto que aparece nas páginas. Você aprenderá sobre:

  • Marcação estrutural: os elementos que você pode usar para descrever títulos e parágrafos.
  • Marcação semântica: que fornece informações extras; como onde a ênfase é colocada em uma frase, que algo que você escreveu é uma citação (e quem o disse), o significado de siglas e assim por diante.

Títulos

<h1>, <h2>, <h3>, <h4>, <h5> e <h6>

O HTML tem seis níveis de títulos:

<h1> é usado para títulos principais

<h2> é usado para subtítulos

Se houver mais seções sob os subtítulos, o elemento <h3> será usado, e assim por diante...

Os navegadores exibem o conteúdo dos títulos em tamanhos diferentes. O conteúdo de um elemento <h1> é o maior, e o conteúdo de um elemento <h6> é o menor. O tamanho exato em que cada navegador mostra os títulos podem variar um pouco. Os usuários também podem ajustar o tamanho do texto em seus navegadores. Você verá como controlar o tamanho do texto, sua cor e as fontes usadas quando chegarmos ao CSS.

Parágrafos

p

Para criar um parágrafo, coloque as palavras que compõem o parágrafo entre uma tag de abertura <p> e uma tag de fechamento </p>.

Por padrão, um navegador mostrará cada parágrafo em uma nova linha com algum espaço entre ele e qualquer parágrafo subsequente.

Negrito e itálico

<b>

Ao colocar palavras entre as tags <b> e </b>, podemos fazer com que os caracteres apareçam em negrito.

O elemento <b> também representa uma seção de texto que seria apresentada de uma maneira visualmente diferente (por exemplo, palavras-chave de um parágrafo), embora o uso do elemento <b> não implique nenhum significado adicional.

<i>

Ao colocar palavras entre as tags <i> e </i>, podemos fazer com que os caracteres apareçam em itálico.

O elemento <i> também representa uma seção de texto que seria dita de uma maneira diferente do conteúdo ao redor - como termos técnicos, nomes de navios, palavras estrangeiras, pensamentos ou outros termos que normalmente seriam colocados em itálico.

Sobrescrito e subscrito

<sup>

O elemento <sup> é usado para conter caracteres que devem ser sobrescritos, como sufixos de datas ou conceitos matemáticos, como elevar um número a uma potência como 22.

<sub>

O elemento <sub> é usado para conter caracteres que devem ser subscritos. É comumente usado com notas de rodapé ou fórmulas químicas como H2O.

Espaço em branco

Para tornar o código mais fácil de ler, os autores de páginas da web geralmente adicionam espaços extras ou iniciam alguns elementos em novas linhas.

Quando o navegador encontra um ou mais espaços um ao lado do outro, ele exibe apenas um espaço. Da mesma forma, se ele encontrar uma quebra de linha, ele a trata como um único espaço também. Isso é conhecido como colápso de espaço em branco.

Você verá frequentemente que os autores de páginas da web aproveitam o colápso de espaços em brando para recuar seu código, a fim de torná-lo mais fácil de ler.

Quebras de linhas e

<b>

Como você já viu, o navegador mostrará automaticamente cada novo parágrafo ou título em uma nova linha. Mas se você quiser adicionar uma quebra de linha no meio de um parágrafo, você pode usar a tag de quebra de linha <b>.

<hr>

Para criar uma pausa entre temas, como uma mudança de tópico em um livro ou uma nova cena em uma peça, você pode adicionar uma régua horizontal entre as seções usando a tag <hr>.

Existem alguns elementos que não têm nenhuma palavra entre a tag de abertura e fechamento. Eles são conhecidos como elementos vazios e são escritos de forma diferente.

Um elemento vazio geralmente tem somente uma tag. Antes do colchete angular de fechamento de um elemento vazio, geralmente haverá um espaço e um caractere de barra.

Marcação semântica

Existem alguns elementos de texto que não tem a intenção de afetar a estrutura das páginas da web, mas eles adicionam informações extras às páginas e são conhecidos como marcação semântica.

No restante do capítulo, você conhecerá mais alguns elementos que ajudarão você a adicionar texto a páginas da web. Por exemplo, você conhecerá o elemento <em> que permite indicar onde a ênfase deve ser colocada em palavras selecionadas e o elemento <blockquote> que indica que um bloco de texto é uma citação.

Os navegadores exibem geralmente o conteúdo desses elementos de uma maneira diferente. Por exemplo, o conteúdo do elemento <em> é mostrado em itálico, e um <blockquote> geralmente é recuado. Mas você não deve usá-los para mudar a aparência do seu texto, o propósito deles é descrever o conteúdo das suas páginas da web com mais precisão.

O motivo para usar esses elementos é que outros programas, como leitores de tela ou mecanismos de busca, podem usar essas informações extras. Por exemplo, a voz de um leitor de tela pode dar ênfase às palavras dentro do elemento <em>, ou um mecanismo de busca pode registrar que sua página apresenta uma citação se você usar o elemento <blockquote>.

Forte e ênfase

<strong>

O uso do elemento <strong> indica que seu conteúdo tem forte importância. Por exemplo, as palavras contidas neste elemento podem ser ditas com forte ênfase.

Por padrão, os navegadores mostrarão o conteúdo de um elemento <strong> em negrito.

<em>

O elemento <em> indica ênfase que altera sutilmente o significado de uma frase.

Por padrão, os navegadores mostrarão o conteúdo de um elemento <em> em itálico.

Citações

Existem dois elementos comumente usados para marcar citações:

<blockquote>

O elemento <blockquote> é usado para citações mais longas que ocupam um parágrafo inteiro. Note como o elemento <p> ainda é usado dentro do elemento <blockquote>.

Os navegadores tendem a recuar o conteúdo do elemento <blockquote>, no entanto, você não deve usar este elemento apenas para recuar um pedaço de texto. Em vez disso, você deve obter esse efeito usando CSS

<q>

O elemento <q> é usado para citações mais curtas que ficam dentro de um parágrafo. Os navegadores devem colocar aspas em volta do elemento <q>, mas o Internet Explorer não faz isso, portanto, muitas pessoas evitam usar o elemento <q>.

Ambos os elementos podem usar o atributo cite para indicar de onde vem a citação. Seu valor deve ser uma URL que terá mais informações sobre a fonte da citação.

Abreviações e siglas

<abbr>

Se você usar uma abreviação ou acrônimo, então o elemento <abbr> pode ser usado. Um atributo title na tag de abertura é usado para especificar o termo completo.

No HTML 4, havia um elemento acronym separado para siglas. Para soletrar a forma completa da sigla, o atributo title era usado. O HTML 5 utiliza apenas o elemento <abbr> para abreviações e siglas.

Citações e definições

<cite>

Ao referenciar um trabalho, como um livro, filme ou artigo de pesquisa, o elemento <cite> pode ser usado para indicar de onde veio a citação.

No HTML 5, <cite> não deveria ser usado para nome de uma pessoa - mas isso era permitido no HTML 4, então a maioria das pessoas continuará usando.

Os navegadores renderizarão o conteúdo de um elemento <cite> em itálico.

<dfn>

A primeira vez que você explica uma terminologia nova (talvez um conceito acadêmico ou algum jargão) em um documento, ela é conhecida como a instância definidora dela.

O elemento <dfn> é usado para indicar a instância definidora de um novo termo.

Alguns navegadores mostram o conteúdo do elemento <dfn> em itálico.

Detalhes do autor

<address>

O elemento <address> tem um uso bastante específico: conter detalhes de contato do autor da página.

Pode conter um endereço físico, mas não necessariamente. Por exemplo, também pode conter um número de telefone ou endereço de e-mail.

Os navegadores geralmente exibem o conteúdo do elemento <address> em itálico.

Você também pode estar interessado em algo chamado microformato hCard para adicionar informações de endereço físico à sua marcação.

Mudanças no conteúdo

<ins> e <del>

O elemento <ins> pode ser usado para mostrar o conteúdo inserido em um documento, enquanto o elemento <del> pode mostrar o conteúdo removido dele.

O conteúdo de um elemento <ins> geralmente é sublinhado, enquanto o conteúdo do elemento <del> geralmente é riscado.

<s>

O elemento <s> indica algo que não é mais preciso ou relevante (mas que não deve ser excluído).

Visualmente, o conteúdo de um elemento <s> geralmente será exibido com uma linha no centro.

Versões mais antigas do HTML tinham um elemento <u> para conteúdo sublinhado, mas isso está sendo descontinuado.
